# ChargePet — Project Context for Claude Code

## What this is

A Flutter hackathon demo that shows real phone charging behavior as a narrative — told through a pet that thrives or suffers based on how responsibly the user charged their phone.

**Source data:** 5 real CSV files (`user_1.csv` through `user_8.csv`) with `power_connected`/`power_disconnected` events, battery %, timestamps in +05:30 timezone.

**Architecture:** Precompute script → scored JSON → Flutter app. No runtime CSV parsing.

## Testing

Framework: Dart's built-in `test` package + `flutter_test`.

**Run scoring engine tests first — always.** These are the ground truth for the demo.

```bash
dart test test/scoring_engine_test.dart
```

Key test cases (must all pass before building UI):
- `sessionScore(1, 99)` → `5` (User 7 Day 1 Session 1)
- `sessionScore(99, 100)` → `0` (phantom+redundant+overcharge clamped)
- `sessionScore(25, 70)` → `100` (sweet spot clamped)
- `sessionScore(25, 80)` → `50` (end=80: deduction not bonus — critical boundary)
- `sessionScore(25, 79)` → `100` (end=79: sweet spot + bonuses)
- `sessionScore(17, 80)` → `25` (User 1 Nov 12 showcase session)
- `sessionScore(50, 53)` → `20` (phantom session — capped at 20)

## Scoring formula (final — do not change without updating JOURNEY.md)

```dart
int sessionScore(int startPct, int endPct) {
  int score = 50;

  // End% deductions (tiered exclusive)
  if (endPct == 100) score -= 30;
  else if (endPct >= 90) score -= 20;
  else if (endPct >= 80) score -= 10;

  // Start% deductions (tiered exclusive)
  if (startPct < 15) score -= 25;
  else if (startPct < 20) score -= 15;
  else if (startPct > 85) score -= 15;

  // Sweet spot bonus
  if (startPct >= 20 && startPct <= 50 && endPct >= 60 && endPct < 80) score += 30;

  // Other bonuses
  if (endPct < 80) score += 10;       // STRICT < 80
  if (startPct >= 20) score += 10;

  // Clamp
  score = score.clamp(0, 100);

  // Phantom cap (post-processing)
  if ((endPct - startPct) < 5) score = score < 20 ? score : 20;

  return score;
}
```

## Key design decisions

See `JOURNEY.md` for the full decision log (D1–D20). The most important ones:

- **D9**: Precompute to JSON — no runtime CSV parsing
- **D11**: Cross-midnight sessions attributed to start date
- **D12**: Orphaned events: skip + warn
- **D13**: Rolling health: normalize by available weights (not zero-fill)
- **D14**: Inertia rule REMOVED (was D(prev)×0.95 for no-session days)
- **D15**: Show daily + weekly + monthly scores on PetHomeScreen
- **D18**: Pet state thresholds shifted up (see above)
- **D20**: Parser uses state machine with pendingConnect variable
